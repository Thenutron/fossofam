## Step 3 â€” Add the Neon database (the one-click part)

1. In your new Vercel project, go to the **Storage** tab.

## Step 4 â€” Create the database tables

The app needs its tables created once. Two easy options:

### Option A â€” From your computer (recommended)

1. Pull the database URL locally:

   ```bash
   npx vercel link        # link this folder to your Vercel project
   npx vercel env pull .env.local
   ```

   This writes `DATABASE_URL` into `.env.local`.

2. Push the schema to Neon:

   ```bash
   npm install
   npm run db:push
   ```

   Drizzle creates the `items`, `dinners`, `expenses`, and `household` tables. When it asks, confirm creating the tables.

3. (Optional) Seed the default dinner rotation and household row:

   ```bash
   npm run db:seed
   ```

   *Note: the app also auto-seeds dinners + the household row on first load, so this is optional.*

### Option B â€” Neon SQL editor

If you'd rather not run anything locally, open the database from Vercel's Storage tab â†’ **Open in Neon** â†’ **SQL Editor**, and paste the contents of `drizzle/0000_*.sql` (generated by `npm run db:generate`). Option A is simpler.

## Using it day to day

- **Dashboard** is the landing view: where to go next, tomorrow's dinner, quick "out of" entry, budget at a glance, quick add-expense.
- **Out of / need**: add items as you run low; they auto-route to the right store.
- **Shop & budget**: store-by-store checklist + log each run's total. Hit **Close out week** on Sunday to snapshot the week and start fresh (this feeds the encouraging "last week" banner).
- **Store route**: suggested order with spend targets and fallback logic.
- **3-week cycle**: tap the current week so the budget envelopes (feed week / bulk week) line up.

## Costs

- **Vercel Hobby**: free. A two-person grocery app uses a tiny fraction of the limits.
- **Neon Free**: free. Your data is well under the storage cap.
- The only "gotcha" is Neon's scale-to-zero: after the app sits idle a while, the first load takes ~1 extra second to wake the database. Harmless for this use.
- Keep it **personal use** (no ads, not for sale) to stay within Vercel's Hobby terms.

## Tuning later

- **Store spend targets** live in `lib/data.ts` (`ROUTE_PLAN`). After a couple of real shops, update the numbers to match your actual receipts.
- **Dinner rotation** seed is in `lib/data.ts` (`DEFAULT_DINNERS`). Edit to your real meals, or just edit them live in the app.
- **Staples, idea banks, baking list** are all in `lib/data.ts` too.
